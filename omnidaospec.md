## 4. Market Creation System (Decentralized but Safe)

### 4.1 Market Proposal Flow

**Functionality**: Allow anyone to propose markets while preventing spam and ambiguity.

#### Smart Contract: `ProposalEngine`

**Key functions**:

* `submitProposal(MarketSpec)`
* `validateProposal(MarketSpec)`
* `lockProposalStake(address proposer)`

**Implementation details**:

* `MarketSpec` stored on IPFS, hash stored on-chain
* Proposer stakes GOV tokens (amount dynamic based on reputation)
* Automated checks:

  * Question template validity
  * Resolution time in future
  * Oracle class supported
  * Category allowed

❌ Invalid → auto-reject + partial slashing
✅ Valid → proposal enters DAO queue

---

### 4.2 Market Templates (Critical Safety Layer)

**Functionality**: Reduce governance overhead and oracle ambiguity.

#### Smart Contract: `TemplateRegistry`

**Key functions**:

* `createTemplate(TemplateSpec)` (DAO-only)
* `approveTemplate(uint256 templateId)`
* `validateAgainstTemplate(MarketSpec)`

**TemplateSpec includes**:

* Outcome type (binary, scalar, multi-outcome)
* Oracle class
* Resolution rules
* Edge-case handling

Markets can *only* be created from approved templates.

---

### 4.3 DAO / Council Approval

**Functionality**: Human review where automation ends.

#### Smart Contract: `Governance`

**Key functions**:

* `voteOnProposal(proposalId)`
* `approveProposal(proposalId)`
* `rejectProposal(proposalId)`

**Design choices**:

* Council elected by DAO
* Fast approval windows
* Slashing for approving bad markets
